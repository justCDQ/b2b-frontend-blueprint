# Page Demo Blueprints

This directory contains page-level blueprints for B2B console demos.

Page blueprints are not implementation code. They describe how component rules should be composed into realistic product pages, including layout, data flow, actions, permissions, states, and responsive behavior.

## Purpose

Use page blueprints to:

- Validate whether component rules work together.
- Give AI a concrete page-level generation target.
- Define demo app requirements before writing code.
- Create acceptance criteria for future template implementation.

## Recommended Page Blueprint Structure

Each page should include:

1. Page goal.
2. User roles and permissions.
3. Layout structure.
4. Component composition.
5. Data contract.
6. Query and pagination flow.
7. Actions and operation rules.
8. State handling.
9. Responsive behavior.
10. AI generation requirements.
11. Acceptance criteria.
